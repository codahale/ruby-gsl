Floating number precision

In Ruby there is a single floating number type, corresponding to the double type in C; the Ruby floating numbers are object of the class Float , with precision
Float::EPSILON = DBL_EPSILON, where DBL_EPSILON is a C macro in float.h; if DBL_EPSILON is not defined, Float::EPSILON is 2**-52 = 2.22045e-16
(see numeric.c).

In the examples we use the =~ method for approximate equality of floating numbers, vectors, etc.; it is defined in gsl_util.rb. By default it a =~ b is true if all components of a and b are equal within precision 1.0e-15 ; most of the examples in this manual work with this precision. A few examples hork with less precision; in that case one can use a =~ [b, eps] to check for equality with precision eps.

assert

Most of the examples in the manual make some simple test for equality, using the function 
def assert(x)
 caller(x) if (!x)
end
For instance 
require 'assert'
require 'GSL'
include Math
require 'gsl_util'
x = 0.1
assert acos(cos(x)) =~ x
p acos(cos(x)) - x
assert 10 * acos(cos(x)) !~ 10 * x 
assert 10 * acos(cos(x)) =~ [10 * x, 1.0e-14]
has output -5.55111512312578e-16
